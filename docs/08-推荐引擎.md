# 08 — 推荐引擎（论文核心）

## 整体架构：三路融合
```
用户请求 → RecommendationEngine.recommend(user_id)
              │
              ├── Pipeline A: CF Engine ──────→ {post_id: score}
              │                                      │
              ├── Pipeline B: Graph Engine ───→ {post_id: score}
              │                                      │
              ├── Pipeline C: Semantic Engine → {post_id: score}
              │                                      │
              └── Fusion Engine ←─────────────────────┘
                    │
                    ↓
              排序后 Top-N 推荐列表
```

---

## Pipeline A: Item-Based CF + IUF (`cf_engine.py`)

### 行为评分权重
| 行为类型 | 权重 |
|---------|------|
| browse | 1.0 × min(duration/60, 3.0) | 最大不超过like的权重
| like | 3.0 |
| favorite | 5.0 |
| comment | 4.0 |

同一用户多次交互同一帖子 → 取最大分数

### 离线预计算：物品相似度矩阵
```
1. 构建交互矩阵
   user_items = {user_id: {post_id: score}}
   item_users = {post_id: {user_id_set}}

2. 计算物品对相似度 (IUF加权余弦)
   对每对物品 (i, j):
     common_users = item_users[i] ∩ item_users[j]
     numerator = Σ_{u ∈ common} [1 / log(1 + |items_of_u|)]   ← IUF惩罚活跃用户
     denominator = √(|item_users[i]| × |item_users[j]|)
     sim(i,j) = numerator / denominator

3. 每个物品保留 Top-50 相似物品
4. 存入 Redis sorted set: item_sim:{post_id}, TTL=24h
```

### 在线推荐
```
输入: user_id
1. 获取用户历史: user_ratings = {post_id: score}
2. 对每个历史物品:
     取 Redis 中 Top-20 相似物品
     对每个相似物品 j (未交互过的):
       score[j] += sim(i,j) × user_rating[i]
3. Min-max 归一化到 [0, 1]
返回: Top-200 候选 {post_id: score}
```

## Pipeline B: Neo4j 3跳涟漪传播 (`graph_engine.py`)

### Cypher 查询逻辑
```
Hop1: 用户 → 种子帖子 (直接交互)
  MATCH (u:User {id: $uid})-[r:LIKED|FAVORITED|COMMENTED|BROWSED]->(seed:Post)
  行为权重: FAVORITED=1.5, COMMENTED=1.2, LIKED=1.0, BROWSED=0.3

Hop2: 标签扩展
  (seed)-[:TAGGED_WITH]->(tag)<-[:TAGGED_WITH]-(candidate)
  tag_score = Σ(seed_weight × 0.4)
  排除已交互帖子

Hop3: 社交信号
  (u)-[:FOLLOWS]->(friend)-[r:LIKED|FAVORITED]->(candidate)
  social_score: FAVORITED=0.15, LIKED=0.10

最终得分: raw_score = tag_score + social_signal
```

### 衰减系数
| 跳数 | 来源 | 系数 |
|------|------|------|
| Hop1 | 直接交互 | 行为权重 (0.3~1.5) |
| Hop2 | 标签传播 | × 0.4 |
| Hop3 | 社交推荐 | 0.10~0.15 |

返回: Top-200 候选，min-max 归一化到 [0,1]

## Pipeline C: Embedding + LLM Reranking (`semantic_engine.py`)

### Stage 1: Embedding 相似度
```
1. user_emb = user.interest_embedding  (1024维，加权平均)
   权重: favorite=2.0, like=1.0

2. 对所有有 content_embedding 的帖子:
   sim = cosine_similarity(user_emb, post_emb)
   embedding_score = (sim + 1) / 2    # [-1,1] → [0,1]

3. Min-max 归一化
```

### Stage 2: LLM Reranking (可选)
```
1. 取 Embedding Top-50 候选

2. 对每个候选帖子，调用千问:
   Prompt: "你是推荐系统评分助手。
   用户兴趣画像：{interest_profile}
   候选文章标题：{title}
   候选文章摘要：{summary}
   请评估匹配度，返回0到10的整数评分，只返回数字。"

3. llm_score = response / 10.0, clamp 到 [0,1]
4. combined = 0.6 × emb_score + 0.4 × llm_score
5. Redis 缓存: llm_rel:{user_id}:{post_id}, TTL=3600s
```

### Stage 3: 合并
```
Top-50 候选: final = combined_score (stage2)
其余候选:    final = embedding_score × 0.6 (降权)
Min-max 归一化 → 返回 Top-N
```

## 融合引擎 (`fusion.py`)

### 默认权重
| Pipeline | 权重 |
|----------|------|
| CF (协同过滤) | 0.35 |
| Graph (图谱) | 0.35 |
| Semantic (语义) | 0.30 |

### 融合公式
```
final_score(p) = w_cf × cf_score(p)
               + w_graph × graph_score(p)
               + w_semantic × semantic_score(p)

缺失分数视为 0.0
```

### 输出格式 (`fuse_with_details`)
```python
{
  'post_id': int,
  'final_score': float,     # 融合总分
  'cf_score': float,        # CF 分量
  'graph_score': float,     # Graph 分量
  'semantic_score': float,  # Semantic 分量
}
```

前端可通过 API 参数 `w_cf`, `w_graph`, `w_semantic` 自定义权重。

---

## 关键创新点（答辩要点）

1. **IUF 惩罚**：CF 中用 `1/log(1+|items_of_u|)` 降低活跃用户权重，避免热门偏差
2. **3跳涟漪传播**：图谱中 Hop1(直接)→Hop2(标签扩展)→Hop3(社交信号)，逐层衰减
3. **两阶段语义**：先 Embedding 粗召回，再 LLM 精排，平衡效果与成本
4. **可调权重**：前端滑块实时调节三路权重，支持消融对比演示
5. **LLM 推荐理由**：独立 API 生成自然语言解释，增强可解释性
