# 03 — 推荐页（核心，答辩重点）

## 页面布局 (`RecommendPage.vue`)

```
┌─────────────────────────────────┐
│  控制面板 (el-card)             │
│  ┌─ 选择用户 ─── UserSelector ─┐│
│  ├─ 融合权重 ─── WeightSlider ─┤│
│  ├─ 推荐数量 ─── el-input-num ─┤│
│  └─ [获取推荐] 按钮 ───────────┘│
├─────────────────────────────────┤
│  推荐结果卡片列表               │
│  ┌── RecCard ──────────────────┐│
│  │ 标题          分数 tag      ││
│  │ 摘要 (2行截断)              ││
│  │ ████████████ ScoreBar       ││
│  │ CF 0.71  Graph 0.92  ...   ││
│  │              [推荐理由] btn ││
│  └─────────────────────────────┘│
│  ... 更多卡片                   │
└─────────────────────────────────┘
```

## 核心流程
1. 用户选择 → `UserSelector` 加载全量用户列表 (el-select filterable)
2. 调整权重 → `WeightSlider` 三路滑块联动 (和=100)
3. 点击「获取推荐」→ 调用 `GET /api/recommend/{userId}?w_cf=0.35&w_graph=0.35&w_semantic=0.30&top_n=20`
4. 渲染 `RecCard` 列表 → 每张卡片含 `ScoreBar` 三色分数条
5. 点击「推荐理由」→ `RecReasonDialog` 调用 LLM 生成解释

---

## 组件详解

### UserSelector.vue
- **功能**：下拉选择用户
- **实现**：`el-select filterable`，`onMounted` 加载全量用户 (`getUserList(1, 200)`)
- **数据流**：`v-model` 双向绑定用户 ID

### WeightSlider.vue
- **功能**：三路权重联动调节，保证和始终 = 100
- **实现**：3 个 `el-slider`，拖动一个时按比例分配剩余值给另外两个
- **颜色**：蓝 CF (#409eff) / 绿 Graph (#67c23a) / 橙 Semantic (#e6a23c)
- **关键算法**：
  ```js
  // 拖动 changedIdx 后
  const remaining = 100 - changed
  const otherSum = others.reduce(sum)
  others.forEach(o => o.value = round(o.value / otherSum * remaining))
  // 修正舍入误差
  ```

### ScoreBar.vue
- **功能**：三色横条可视化三路分数占比
- **实现**：CSS flex，三段 `div`，宽度按 `cf:graph:semantic` 比例
- **高度 8px**，圆角 4px，hover 显示具体分值 (title 属性)

### RecCard.vue
- **功能**：推荐结果卡片
- **展示**：标题（可点击跳转详情）、分数 tag、摘要（2行截断）、ScoreBar、三路具体分值、推荐理由按钮
- **布局**：el-card shadow="hover"

### RecReasonDialog.vue
- **功能**：弹窗展示 LLM 推荐理由
- **实现**：`el-dialog`，watch `modelValue` 变化时自动请求 `getRecommendReason`
- **loading 状态**：显示旋转图标 + "正在生成推荐理由..."
