# 問題
- JWT鉴权知识
- 原来的问题：session 登录- server生成session 获取sessionid - 存入客户端cookie - 之后客户端请求带cookie - 服务器验证sessionid识别用户
- JWT相当于把这个session交给你了，服务器每次请求只需要验证客服端发来的JWT即可（每次请求都有这个JWT请求头）
- 不算加密：payload和header只是b64编码可转，签名是防篡改

- vite代理：解决跨域cors问题，只在开发用（前后端分离的开发），浏览器认为请求来自同一源

- 帖子爬取：采用公开的api平台，部分知识社区不允许爬取

- 用户真实偏好和注册偏好不一致：三路融合的捕捉，行为够多能够这个问题。
