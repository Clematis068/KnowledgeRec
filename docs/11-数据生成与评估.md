## init_db.py — 初始化数据库

```python
def init_mysql():
    db.create_all()  # 根据 ORM 模型建表

def init_neo4j():
    # 创建唯一约束
    CREATE CONSTRAINT FOR (u:User) REQUIRE u.id IS UNIQUE
    CREATE CONSTRAINT FOR (p:Post) REQUIRE p.id IS UNIQUE
    CREATE CONSTRAINT FOR (t:Tag) REQUIRE t.id IS UNIQUE
    CREATE CONSTRAINT FOR (d:Domain) REQUIRE d.id IS UNIQUE
```

## generate_mock_data.py — 数据生成

### 数据规模
| 实体 | 标准模式 | `--small` 模式 |
|------|---------|---------------|
| Domain | 10 | 10 |
| Tag | 100 (10/域) | 100 |
| User | 500 | 20 |
| Post | 2000 | 50 |
| Behavior | ~50,000 (100/人) | ~500 |
| Follow | ~3,000 (6/人) | — |

### 10 个知识领域
计算机科学、数学、物理学、哲学、经济学、生物学、心理学、历史学、文学、法学

每个领域 10 个专业标签，如 CS: [机器学习, 深度学习, NLP, CV, 数据库, 操作系统, 算法, 分布式系统, 网络安全, 编程语言]

### 生成流程 (9步)
```
1. generate_domains_and_tags()     # 10域 + 100标签
2. generate_users(n)               # 用户 + 2-4个兴趣域
3. generate_posts(users, n)        # 帖子 + 1-3标签
4. generate_behaviors(users, avg)  # 行为 (60%浏览 20%点赞 10%收藏 10%评论)
5. generate_follows(users, avg=6)  # 关注 (70%兴趣相似 30%随机)
6. generate_embeddings()           # 千问API: 标签+帖子 Embedding
7. compute_user_interest_embeddings() # 用户兴趣向量 (加权平均)
8. generate_interest_profiles(50)  # LLM生成兴趣画像 (采样50人)
9. sync_to_neo4j()                 # 同步全部数据到 Neo4j
```

### 行为分布
- 90% 交互在用户兴趣领域内，10% 随机（模拟偶然浏览）
- browse 时长: 5~300 秒随机
- comment 文本: 从 10 个模板随机选取

### Embedding 生成
- 标签 Embedding: `text-embedding-v3(tag.name)`
- 帖子 Embedding: `text-embedding-v3(title + content[:200])`
- 用户兴趣向量: 加权平均 (favorite=2.0, like=1.0)
- 限速: 每次调用间隔 0.1s，每 20/50 条 checkpoint

## evaluate.py — 消融实验

### 消融配置 (7组)
| 配置名 | w_cf | w_graph | w_semantic |
|--------|------|---------|-----------|
| CF_only | 1.0 | 0.0 | 0.0 |
| Graph_only | 0.0 | 1.0 | 0.0 |
| Semantic_only | 0.0 | 0.0 | 1.0 |
| CF+Graph | 0.5 | 0.5 | 0.0 |
| CF+Semantic | 0.5 | 0.0 | 0.5 |
| Graph+Semantic | 0.0 | 0.5 | 0.5 |
| **Full_Fusion** | **0.35** | **0.35** | **0.30** |

### 评估指标
| 指标 | 公式 | K 值 |
|------|------|------|
| Precision@K | \|rec[:K] ∩ rel\| / K | 5, 10, 20 |
| Recall@K | \|rec[:K] ∩ rel\| / \|rel\| | 5, 10, 20 |
| NDCG@K | DCG/IDCG, DCG = Σ 1/log₂(i+2) | 5, 10, 20 |

### 测试集构建
- **时间切分**: 每个用户最后 20% 的 like/favorite 行为作为测试集
- **过滤**: 仅保留 ≥5 条 like/favorite 的用户
- **结果**: 输出对比表格，展示各配置在不同 K 值下的表现

### 期望结论
Full_Fusion (三路融合) 应在大多数指标上优于单路或两路组合，验证融合策略的有效性。
